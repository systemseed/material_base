Material Base theme folder structure
================

Files and folders of Material Base:

* `ğŸ“ config` - contains default blocks configuration and theme settings
* `ğŸ“ dist` - contains builded assets (CSS, JS, images, SVG icons sprite)
* `ğŸ“ docs` - contains documentation
* `ğŸ“ icons` - source folder for building SVG icons sprite
* `ğŸ“ images` - source folder for CSS used images like backgrounds
* `ğŸ“ js` - source folder for JS files
* `ğŸ“ material_base_mdc` - "Material Base MDC" theme folder
* `ğŸ“ material_base_subtheme` - example kit for creating your custom subtheme
* `ğŸ“ scss` - source folder for Sass files
* `ğŸ“ templates` - contains Twig templates for Drupal elements and theme components
* `ğŸ“„ .gitignore` - contains exclusions for Git repository
* `ğŸ“„ CHANGELOG.txt` - contains development changes log
* `ğŸ“„ composer.json` - contains technical information about theme
* `ğŸ“„ favicon.ico` - theme favicon
* `ğŸ“„ generate.php` - subtheme generation script
* `ğŸ“„ logo.svg` - theme logo
* `ğŸ“„ material_base.breakpoints.yml` - contains Drupal breakpoints definitions
* `ğŸ“„ material_base.info.yml` - contains technical information about theme, used libraries, regions and other options
* `ğŸ“„ material_base.libraries.yml` - contains Drupal libraries definitions
* `ğŸ“„ material_base.theme` - contains functions used in theme
* `ğŸ“„ package.json` - contains technical information about building tools
* `ğŸ“„ README.md` - contains theme description
* `ğŸ“„ screenshot.svg` - theme screenshot
* `ğŸ“„ theme-settings.php` - contains theme settings definion
* `ğŸ“„ webpack.config.js` - contains build tasks configuration
* `ğŸ“„ yarn.lock` - contains technical information about installed versions of build tools

Pretty much the same structure could be used for your custom theme but not including `docs`, `material_base_mdc` and `material_base_subtheme` folders.
